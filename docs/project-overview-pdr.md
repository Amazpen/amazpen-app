## סקירה כללית

**המצפן** (Amazpen) הוא מערכת SaaS לניהול עסקי, מותאמת עברית ו-RTL, המיועדת בעיקר למסעדות ועסקים קטנים-בינוניים. המערכת מספקת כלים למעקב אחר מדדים פיננסיים, הוצאות, יעדים, ספקים ותובנות מבוססות בינה מלאכותית.

### פרטים טכניים מרכזיים

| פרמטר | ערך |
|--------|------|
| **פריימוורק** | Next.js 16 (App Router), React 19, TypeScript 5.9 (strict) |
| **עיצוב** | Tailwind CSS 4, ערכת נושא כהה (dark theme), RTL |
| **UI** | Radix UI + shadcn/ui (סגנון new-york), Lucide icons |
| **מסד נתונים** | Supabase (PostgreSQL, Auth, Realtime, Storage) |
| **AI** | OpenAI GPT-4.1-mini דרך Vercel AI SDK, תגובות streaming |
| **פריסה** | Docker multi-stage → Dokploy/Traefik, פורט 3001 |
| **סוג אפליקציה** | PWA (Progressive Web App) |

## חזון המוצר

המצפן נבנה עם החזון לספק לבעלי עסקים כלי ניהול פיננסי מקיף, חכם ונגיש — בעברית מלאה. המערכת משלבת:

- **מעקב פיננסי יומי** — הזנת נתוני הכנסות והוצאות יומיים
- **ניהול ספקים מלא** — חשבוניות, תשלומים, יתרות, מעקב מחירים
- **תובנות AI** — צ'אט חכם שמנתח את הנתונים ומציע פעולות
- **סריקת מסמכים** — OCR אוטומטי לחשבוניות וקבלות
- **יעדים ובונוסים** — הגדרת יעדים ותוכניות תגמול לעובדים
- **דוחות** — ניתוח תזרים מזומנים, דוחות חודשיים, מעקב תקציב

## מפת תכונות

### 1. דשבורד ראשי (`/`)
- סיכום פיננסי חודשי — הכנסות, הוצאות, רווח
- גרפים אינטראקטיביים (Recharts) — מגמות הכנסות, התפלגות הוצאות
- בחירת עסקים מרובים (multi-business selection)
- מדדים מרכזיים (KPIs) עם אייקונים ייחודיים

### 2. מילוי יומי (`/daily-entries` — מודאל)
- הזנת הכנסות יומיות לפי מקורות הכנסה
- תמיכה ב-day_factor (ימים חלקיים: 0–1)
- לוח זמנים שבועי (`business_schedule`) עם חריגים (`business_day_exceptions`)
- ממשק טפסים עם טיוטה אוטומטית (useFormDraft)

### 3. הוצאות וחשבוניות (`/expenses`)
- טבלת חשבוניות עם מיון, סינון וחיפוש גלובלי
- הוספה, עריכה ומחיקה של חשבוניות
- קישור חשבוניות לספקים
- הנחות — גלובלית או לפי פריט (₪/%)
- מודאל חשבונית מאוחדת (ConsolidatedInvoiceModal)
- טאבים: חשבוניות, תשלומים, יתרות ספקים

### 4. תשלומים (`/payments`)
- רישום תשלומים לספקים
- קישור תשלום לחשבונית
- מעקב יתרות ספקים
- התראות על תשלומים קרובים

### 5. ספקים (`/suppliers`)
- ניהול ספקים — פרטים, אנשי קשר, תנאי תשלום
- אחוז הנחה ברירת מחדל לכל ספק
- מעקב תקציב ספק
- היסטוריית חשבוניות ותשלומים

### 6. מעקב מחירים (`/price-tracking`)
- השוואת מחירים בין ספקים לאורך זמן
- זיהוי שינויי מחירים ומגמות
- התראות על עליות מחיר חריגות

### 7. יעדים (`/goals`)
- הגדרת יעדים חודשיים ושנתיים
- מעקב התקדמות בזמן אמת
- השוואה בין יעד לביצוע בפועל

### 8. תוכניות בונוס (`/bonus-plans` — admin)
- הגדרת תוכניות תגמול לעובדים
- חישוב בונוסים אוטומטי לפי ביצועים
- תמיכה בנוסחאות מותאמות אישית

### 9. לקוחות (`/customers`)
- ניהול לקוחות עסקיים
- מעקב הכנסות לפי לקוח
- ערוצי קליטה (intake channels)

### 10. תזרים מזומנים (`/cashflow`)
- תחזית תזרים מזומנים
- ניתוח הכנסות מול הוצאות צפויות
- הוצאות חוזרות (recurring expenses)
- התחייבויות קודמות (prior commitments)

### 11. דוחות (`/reports`)
- דוחות חודשיים ושנתיים
- ניתוח רווחיות
- ייצוא לפורמטים שונים

### 12. צ'אט AI (`/ai`)
- ממשק שיחה חכם עם סוכן AI
- גישה לנתוני העסק בזמן אמת
- כלים: סיכום חודשי, שאילתות SQL, לוח זמנים, יעדים, מחשבון
- הצעת פעולות (proposeAction) עם אישור משתמש
- שמירת היסטוריית שיחות

### 13. סריקת מסמכים — OCR (`/ocr`)
- העלאת חשבוניות וקבלות (תמונות + PDF)
- עיבוד אוטומטי באמצעות OpenAI Vision
- תור מסמכים (DocumentQueue) עם מצב: ממתין, מעובד, מאושר, נדחה
- עורך פריטים ידני — הוספה ומחיקה של שורות
- מילוי אוטומטי של הנחת ספק ברירת מחדל
- צפייה במסמכים (DocumentViewer) כולל תמיכה ב-PDF

### 14. תובנות (`/insights`)
- ניתוח חכם של נתוני העסק
- זיהוי מגמות ואנומליות
- המלצות לשיפור ביצועים

### 15. הגדרות (`/settings`)
- ניהול פרופיל עסקי
- לוח זמנים שבועי
- מקורות הכנסה
- אמצעי תשלום
- ניהול משתמשים וגישה

### 16. ניהול מערכת — Admin (`/admin`)
- ניהול משתמשים ועסקים
- ייבוא נתונים היסטוריים
- סקירת הנהלת חשבונות
- תוכניות בונוס
- תקציבי ספקים
- ניהול יעדים
- ניהול הוצאות ותשלומים
- חריגי ימים
- ייבוא יעדים
- משתמשים מחוברים (online users)
- התחייבויות

## ארכיטקטורה טכנית

### שכבות המערכת

```
┌─────────────────────────────────┐
│         PWA (Service Worker)    │
├─────────────────────────────────┤
│    Next.js 16 — App Router     │
│  ┌───────────┬───────────────┐  │
│  │  (auth)   │ (dashboard)   │  │
│  │  Login    │ כל הדפים      │  │
│  │  Forgot   │ העסקיים       │  │
│  └───────────┴───────────────┘  │
├─────────────────────────────────┤
│         API Routes (19)         │
│  AI, Upload, Push, Admin, ...   │
├─────────────────────────────────┤
│     Supabase (PostgreSQL)       │
│  Auth │ Realtime │ Storage      │
└─────────────────────────────────┘
```

### Route Groups

| קבוצה | תיאור | דוגמאות |
|--------|--------|---------|
| `(auth)` | דפים ציבוריים — אימות | login, forgot-password, logout |
| `(dashboard)` | דפים מוגנים — עסקיים | expenses, suppliers, ai, ocr, admin |
| `api/` | נקודות קצה של API | ai/chat, upload, push, health |

### DashboardContext — ניהול State מרכזי

ה-state הראשי של האפליקציה מוגדר ב-`src/app/(dashboard)/layout.tsx` ומספק:

- `selectedBusinesses` — מצב בחירת עסקים מרובים
- `isAdmin` — בדיקת תפקיד Admin
- `refreshProfile()` — רענון נתוני משתמש/עסק

גישה דרך `useDashboard()` hook.

## אבטחה והרשאות

### אימות (Authentication)

- **Supabase Auth** — email/password + Google OAuth
- **Middleware** (`middleware.ts`) — בדיקת אימות בכל בקשה
- ניתוב אוטומטי ל-`/login` למשתמשים לא מאומתים
- נתיבי API מוחרגים מבדיקת middleware

### הרשאות (Authorization)

- **Row Level Security (RLS)** — מדיניות ברמת שורה ב-Supabase
- **תפקידים**: admin, owner, employee
- **בדיקת תפקיד בצד שרת** — `apiAuth.ts` לאימות API routes
- **בדיקת תפקיד בצד לקוח** — `isAdmin` מ-DashboardContext

### הגנות נוספות

- Rate limiting — 20 בקשות/דקה לצ'אט AI
- שאילתות SQL בצ'אט AI — קריאה בלבד (read-only)
- הצפנת תעבורה — HTTPS דרך Traefik
- VAPID keys להתראות Push

## מערכת AI

- **מודל**: OpenAI GPT-4.1-mini

## מערכת OCR

### תהליך סריקה

1. **העלאת מסמך** — תמונה או PDF דרך `/api/upload`
2. **המרת PDF** — `pdfToImage.ts` ממיר PDF לתמונות
3. **עיבוד OCR** — OpenAI Vision דרך `/api/ai/ocr`
4. **טופס עריכה** — `OCRForm` להשלמה ותיקון ידני
5. **אישור ושמירה** — יצירת חשבונית אוטומטית

### רכיבי OCR

- `DocumentQueue` — תור מסמכים ממתינים
- `DocumentViewer` — צפייה במסמך (כולל PDF)
- `OCRForm` — טופס עריכת נתוני OCR

### תכונות

- הוספה ומחיקה ידנית של שורות פריטים
- מילוי אוטומטי של הנחת ספק ברירת מחדל
- סימון סוגי מסמכים בצבעים
- ארכיון מסמכים מעובדים

## PWA ותמיכה אופליין

### Service Worker

- `public/sw.js` — ניהול cache עם build timestamp
- Cache busting אוטומטי דרך `next.config.ts`
- `public/manifest.json` — הגדרות PWA

### תמיכה אופליין

- **IndexedDB** — `idb-keyval` לאחסון מקומי
- **useOfflineSync** — hook לסנכרון נתונים אופליין
- **OfflineIndicator** — חיווי מצב רשת

### התראות Push

- **web-push** — שליחת התראות מהשרת
- **VAPID keys** — אימות התראות
- `/api/push/` — נקודות קצה לניהול התראות
- `/api/daily-push/` — התראות יומיות
- `/api/bonus-push/` — התראות בונוס

### hooks קשורים

- `usePushSubscription` — ניהול מנוי להתראות
- `useWakeLock` — שמירה על מסך דלוק

## פריסה ותשתית

### Docker

- **בנייה multi-stage**: deps → builder → runner
- **Base image**: `node:20-alpine`
- **פורט**: 3001 (להימנעות מקונפליקט עם Dokploy)
- **output**: `standalone` ב-`next.config.ts`

### Deployment Pipeline

```
Git Push → Docker Build → Dokploy → Traefik (Reverse Proxy) → HTTPS
```

### פקודות

```bash
npm run dev              # שרת פיתוח (פורט 3000, Turbopack)
npm run dev:webpack      # שרת פיתוח ללא Turbopack
npm run build            # בנייה לפרודקשן
npm start                # הפעלת שרת פרודקשן
npm run lint             # בדיקת ESLint
```

### משתני סביבה

| משתנה | סוג | תיאור |
|--------|------|--------|
| `NEXT_PUBLIC_SUPABASE_URL` | Build-time | כתובת Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Build-time | מפתח ציבורי Supabase |
| `NEXT_PUBLIC_DISABLE_REALTIME` | Build-time | ביטול Realtime (false) |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | Build-time | מפתח VAPID להתראות |
| `SUPABASE_SERVICE_ROLE_KEY` | Runtime | מפתח שירות Supabase |
| `OPENAI_API_KEY` | Runtime | מפתח OpenAI |

## דרישות ביצועים

### זמני טעינה

- First Contentful Paint: < 1.5 שניות
- Time to Interactive: < 3 שניות
- Lazy loading לגרפים (Recharts)

### Realtime

- Supabase Realtime עם presence tracking
- `useRealtimeSubscription` ו-`useMultiTableRealtime` hooks
- ניתן לביטול עם `NEXT_PUBLIC_DISABLE_REALTIME=true`

### אופטימיזציות

- Turbopack לפיתוח מהיר
- Standalone output לבנייה קומפקטית
- Service Worker caching
- IndexedDB לנתונים אופליין
