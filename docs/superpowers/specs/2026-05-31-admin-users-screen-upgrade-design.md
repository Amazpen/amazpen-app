# שדרוג מסך ניהול משתמשים (Admin Users)

תאריך: 2026-05-31
קובץ יעד: `src/app/(dashboard)/admin/users/page.tsx`

## מטרה

שדרוג כולל של מסך ניהול המשתמשים ב-3 צירים: פונקציונליות (חיפוש/מיון/ספירה),
העשרת המידע המוצג בכרטיס, והתאמה לסטנדרט העיצוב של מסכי ה-admin — בלי לשבור את
ה-realtime, ה-RLS, או flow היצירה/עריכה הקיימים.

## גישה

שיפור **in-place** של המסך הקיים (לא בנייה מחדש כטבלה). המסך הוא כרטיסים
מותאמי-מובייל/PWA; מבנה הכרטיסים נשמר ומועשר. מוסיפים שכבת בקרה (חיפוש/מיון/ספירה)
מעל הרשימה, ומעשירים את הכרטיס וה-header.

## אימות סכמה (בוצע מול ה-DB)

- `profiles`: `id, email, full_name, phone, avatar_url, is_admin, created_at, deleted_at` — קיים.
- `business_members`: `id, business_id, user_id, role, invited_at, joined_at, created_at, deleted_at` — קיים.
- `businesses`: `id, name, deleted_at` — קיים.
- **Gotcha:** `business_members` ו-`businesses` הן soft-delete (`deleted_at`). שאילתת הצ'יפים חייבת לסנן `deleted_at IS NULL` בשתי הטבלאות.

## שינויים מפורטים

### 1. Header — התאמה לסטנדרט admin
- נשמר: עיגול `#4A56D4` 60×60 + אייקון users + כותרת `text-[24px] font-bold` ממורכז.
- **נוסף:** תת-כותרת `text-[14px] text-white/50` ממורכזת: "ניהול משתמשים, הרשאות ושיוך לעסקים".

### 2. סרגל בקרה חדש (מתחת ל-Header, מעל כפתור ההוספה)
- **חיפוש (client-side):** `Input` עם אייקון זכוכית מגדלת מימין (ראשון ב-DOM = ימין ב-RTL) וכפתור X לניקוי כשיש טקסט. מסנן את הרשימה הנטענת לפי `full_name` / `email` / `phone` (case-insensitive, `includes`). חל גם על מצב "כל המשתמשים" וגם על מצב עסק ספציפי.
- **מיון (`Select`):** אפשרויות — "הצטרפו לאחרונה" (ברירת מחדל, `created_at`/`joined_at` יורד), "שם (א-ב)", "תפקיד" (במצב עסק). נשמר ב-`usePersistedState` כמו הסינון הקיים.
- **ספירה:** טקסט `text-[13px] text-white/50` — "X משתמשים" (מתעדכן לפי תוצאות הסינון/חיפוש).

החיפוש והמיון מיושמים על הנתונים שכבר ב-state (`allUsers` / `businessUsers`) דרך
`useMemo` — בלי שאילתות נוספות, בלי לשבור realtime.

### 3. כרטיס משתמש מועשר
- **טלפון:** אם `phone` קיים — שורה עם אייקון טלפון מתחת לאימייל, עטוף `dir="ltr"` כדי שהמספר יוצג נכון ב-RTL.
- **תאריך הצטרפות:** טקסט קטן `text-[11px] text-white/40` — `created_at` במצב all, `joined_at` במצב עסק. פורמט `he-IL` קצר. אם אין `joined_at` (ממתין) — לא מוצג תאריך.
- **צ'יפים של עסקים (מצב "כל המשתמשים" בלבד):** לכל משתמש צ'יפים קטנים עם שם העסק + תפקיד. נטען דרך `useMultiTableRealtime`-friendly fetch נוסף.

### 4. שאילתת שיוך-עסקים (למצב "כל המשתמשים")
- שאילתה אחת: `business_members` עם `businesses(id, name, deleted_at)`, מסונן `business_members.deleted_at IS NULL`.
- בצד הלקוח: מסננים גם `businesses.deleted_at IS NULL`, ובונים map `user_id → [{ businessName, role }]`.
- נטען רק כשמצב הסינון הוא `all` (יחד עם `fetchUsers`), ומתרענן ב-realtime עם שאר הנתונים.
- חדש state: `membershipsByUser: Record<string, {businessName, role}[]>`.

### 5. מצבי טעינה וריקות
- **Skeleton:** בזמן fetch ראשוני של הרשימה — 3-4 כרטיסי-שלד אפורים (פולסציה) במקום ריק.
- **Empty-state לחיפוש:** כשהחיפוש לא מחזיר תוצאות — הודעה נפרדת "לא נמצאו משתמשים התואמים לחיפוש" (שונה מ"אין משתמשים במערכת").

## אילוצים ושמירה על קיים

- **RTL:** אייקון חיפוש ראשון ב-DOM (ימין). מספרי טלפון ב-`dir="ltr"`. צ'יפים `flex-wrap`.
- **לא לגעת ב-RLS** — קריאה בלבד דרך ה-client הקיים; `is_admin` כבר נבדק.
- **realtime** קיים (`profiles`, `business_members`) נשמר; שאילתת הצ'יפים נכנסת לאותו `fetchUsers`.
- **דיאלוגים** (יצירה/עריכה) — לא משתנים לוגית; ליטוש ויזואלי קל בלבד אם נדרש, ללא שינוי flow.
- **YAGNI:** בלי pagination/וירטואליזציה (כמות המשתמשים סבירה), בלי export, בלי bulk-actions.

## בדיקות (ידני, browse/PWA)

1. חיפוש לפי שם/אימייל/טלפון מסנן נכון בשני המצבים.
2. מיון משנה סדר ונשמר אחרי refresh.
3. ספירה תואמת למספר המוצג.
4. צ'יפים מציגים רק עסקים פעילים (לא מחוקים).
5. טלפון מוצג LTR תקין; תאריך בפורמט he-IL.
6. Skeleton בזמן טעינה; empty-state לחיפוש ריק.
7. יצירה/עריכה/מחיקה/שינוי-תפקיד עדיין עובדים כרגיל.
